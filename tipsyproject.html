A local bartending service based out of Los Angeles, Tipsy Bar provides open bar services for partys and special events. As a service based company they need a website that is able to inform the user about the type of services and experience they provide and while also being able to handle form requests from prospective clients.

A Static Mobile First Webpage

Tipsy Bars main focus was to have a website that represented the brand well while also having the ability to process form data. With the main objective being to inform people of the services and handle form requests, this type of website is very straight forward to build, as no user has the need for dynamic content. Even with this in mind I decided to code all of the back-end myself instead relying on a third party.

Responsive For Every Device

One of the most important aspects of web design is being able to design a website to render well on a variety of devices, from the largest desktop screens to the small screens of any smartphone. Responsive design ensures the user has a consistent experience no matter what device they browse on.


With the emergence of mobile browsing making up more than 50% of all web browsing, I decided to take a mobile first approach instead of the common approach of initially designing for the desktop. The process was a little diffrent considering the compact nature of mobile design so lets see how it all went down.

Streamlined and To the Point

Every Project I undertake is unique with its own purpose and goals. Here I highlight a quick break down of the main steps I took to develop the Tipsy Bar website.

phase — 01

Initial
Meeting

The first meeting with a client goes over the main objective and goals of the site. Once I have a definitive idea of what the website is suppost to do then i move to the next phase.

phase — 02

Design
Stage

This is where the real fun begins. This stage of development is where I get my hands dirty with design(no code just yet!). The main tools I use for designing is adobe XD and illustrator. Being able to create and edit backgrounds and svgs makes the creative process in XD more streamlined.

phase — 03

Development
Stage

Now the coding begins! For Tipsy Bar my main focus was to build a Static site that can also handle form data through the use of our own RESTful routing. Once we have our form data stored on our MongoDb, we want to be able to display the form data to authenticated users on our own show page.

Everything you need.
Nothing you don't.

Building the Tipsy Bars front end and back end made it more imperative that I only added the features we needed instead of features that were non-essential and just take up processing power, so we can end up with a lightweight yet functional website.
